# Zippy — UX/UI (Visual DNA)

## 1) ADN visual
**“Claro · minimal · profesional”**

Objetivo: interfaces limpias, decisiones rápidas y baja fricción operativa.

## 2) Reglas generales
- Jerarquía visual fuerte: títulos claros, acciones primarias evidentes.
- Evitar ruido: una acción principal por bloque.
- Consistencia transversal entre Passenger y Admin.
- Accesibilidad base: contraste, foco visible, navegación teclado.

## 3) Lenguaje de producto
- **Código en inglés** (componentes, props, variables).
- **UI en español (Argentina)**: labels, mensajes, validaciones, notificaciones.
- Tono: directo, cordial, profesional.

## 4) Componentes base (MVP)
- Botones: primario, secundario, ghost.
- Inputs: texto, selección, fecha/hora, búsqueda.
- Feedback: alertas, badges de estado, toasts.
- Data display: tablas simples, cards de resumen, timelines de estado.
- Navegación: top bar + menú lateral (Admin).

## 5) Espaciado y layout
- Grid base de 8px.
- Espaciados recomendados: 8 / 12 / 16 / 24 / 32.
- Contenedores con padding consistente y aire visual.
- Formularios: bloques cortos, validación inline clara.

## 6) Estados y microinteracciones
- Estados obligatorios: loading, vacío, error, éxito.
- Mensajes accionables (“Qué pasó” + “Qué hacer”).
- Confirmaciones explícitas para acciones destructivas.

## 7) Criterios de calidad UX
- Flujo crítico resoluble en pocos pasos.
- Terminología consistente en toda la plataforma.
- Tiempo de aprendizaje bajo para operación diaria.
